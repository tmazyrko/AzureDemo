Pylint

Pylint is a code, bug and quality checker for the Python programming language that tries to enforce a coding standard and looks for code smells.


Getting Started

Installing Pylint is simple, just use pip:

pip install pylint

To lint your code, invoke Pylint from the command line. The usage is:

pylint [options] modules_or_packages

You should give Pylint the name of a python package or module, or some number of packages or modules, but it is also possible to analyze Python files (with a few restrictions). The thing to keep in mind is that Pylint will try to convert the file name to a module name, and only be able to process the file if it succeeds.

pylint mymodule.py

The above should always work since the current working directory is automatically added on top of the python path.

pylint directory/mymodule.py

The above will work if ‘directory’ is a python package (i.e. has an __init__.py file or it is an implicit namespace package) or if "directory" is in the python path.

For more help with using Pylint, refer to the documentation here.


Tip: Pytest Plugin

Pytest is a full-featured Python testing tool/framework that allows you to write and run a suite of unit tests against your code. If you are already using Pytest for a Python project, you can download the pytest-pylint plugin and run pylint in conjunction with your unit tests so that any pylint errors will fail the build as well (configurable by severity).

After installing Pytest, run the following pip command:

pip install pytest-pylint

Then, simply run Pytest from the command line like so:

pytest --pylint


Tip: Pytest Plugin

What Pylint says about your code should not be taken about gospel, and it’s entirely possible that it will warn you about things that you have conscientiously done. There are a few ways to deal with pylint’s verboseness, such as creating a custom configuration file (explained here).

However, a quick and dirty way of ignoring Pylint errors is to simply add a comment to a problematic class/function with the following format:

# pylint: disable=<flags, comma-delimited>
