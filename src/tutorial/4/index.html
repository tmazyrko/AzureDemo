Pushing your code to Github


Create a repository on Github

Before we can push files to a repository, we need to create said repository. Head to github.com, log in, click the ‘+’ in the top right corner, and select ‘New repository’. Give your new repository a name and create it (you don’t need to enable any of the other options).


Initialize the Repository on your Local Machine

We must also turn our local project directory into a local repository to tell git to start tracking files in our project. Run the following command in your project root directory to do so:

git init


Make a Commit

A git commit is a snapshot of your repo at a specific point in time, but before we can make a commit we must stage changes, or specify which changed files we want to include in it. Use the git add command:

git add .

The period indicates that we are staging all of the files/folders in this directory, rather than any particular ones. Next, make the commit:

git commit -m “Initial commit”

The -m flag allows us to add a descriptive message of what the commit changed - you should always add a message to your commits!


Push your commits to GitHub

The final step is to link our local repository to the one we created on GitHub. Let’s name our branch ‘main’ and add the GitHub repo as a remote named ‘origin’:

git branch -M main
git remote add origin [GitHub repo URL].git

Replace the bracketed text with the URL to your created GitHub repo, and make sure to append a ‘.git’ to it. Finally, push your commit to GitHub!

git push -u origin main
