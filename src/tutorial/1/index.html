Prerequisites


Install Docker

Docker is an industry-standard tool that uses OS-level virtualization to deliver software in packages called containers. Containers are isolated from one another and bundle their own software, libraries, and configuration files. We will be using a Docker container to spin up an instance of Apache web server, and then later we will take advantage of Docker’s integration with Microsoft Azure to deploy that container directly to the cloud.

To get started with Docker, simply go to the Docker website and download and install the correct version of Docker Desktop for your system. This will install the Docker application, as well as Docker CLI tools.

While you’re at it, sign up for a Docker Hub account as we will need it later.


Install git

Git is a software that can track changes in a specified set of files, usually for the purpose of coordinating work among collaborating programmers and keeping track of codebase versions as development progresses. Git can be used with privately hosted repositories as well as with public services such as GitHub, BitBucket, Gitea, etc. For the purposes of this tutorial we will be using GitHub as our hosting provider, so if you do not have an account with GitHub (or an alternative) go ahead and create one now.

To get started with git, head to the git project website and download and install the correct client for your operating system.
