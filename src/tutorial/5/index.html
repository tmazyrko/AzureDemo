Build a Docker image and push it to Docker Hub


Build the image

A Docker image contains everything needed to run a containerized application, including code, config files, environment variables, libraries and runtimes. When the image is deployed to a Docker environment, it can be executed as a Docker container. In our case, we need to package our website up into an image that Azure can then download and use to create an instance of our container.

docker build -t username/azurewebsite .

To be able to upload the image to Docker Hub, we must name it correctly - the naming convention is [username]/[repo name]:[optional tag] so make sure you fill in your Docker Hub username when running the build command. We can omit the optional tag for the image, Docker will just assign the default tag of “latest”. Once the image is built, we can upload it to Docker Hub with the following command:

docker push username/azurewebsite

With the image pushed, we are now ready to deploy our website to Azure!
